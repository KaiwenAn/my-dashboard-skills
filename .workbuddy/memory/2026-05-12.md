## 重大修复：describe_table 和 fetch_results 方法

### 问题
用户期望的 skill 用法是：用户通过自然语言输入表名后，skill 能自动获取表详情（字段名、类型、注释），然后自主判断需要哪些字段。

但 `describe_table` 方法一直无法获取到字段信息。

### 根因分析
经过深入调试，发现有两个问题：

1. **`fetch_results` 方法的请求格式错误**：
   - 原代码：`resp = self.session.post(url, json=payload, headers=...)`
   - 问题：queryId 作为 JSON body 传递，导致 404 错误
   - 修复：改为 `resp = self.session.post(url, params=payload, headers=...)`，queryId 作为 URL 参数传递

2. **`execute_query` 对非 SELECT/WITH 语句不获取结果**：
   - 原代码：`if sql.strip().upper().startswith("SELECT") or sql.strip().upper().startswith("WITH")`
   - 问题：`DESCRIBE` 语句不以 "SELECT" 开头，所以不调用 `fetch_results`
   - 修复：在 `describe_table` 中直接调用底层方法（`submit_query` + `wait_for_completion` + `fetch_results`）

### 修改文件

### 修复后的效果
- `describe_table` 方法能正确获取表字段信息
- 测试表：`iceberg_zjyprc_hadoop.meta.dwd_user_module_page_view`
- 获取到 18 个字段，包括：date, uid, e_ts, session_id, module, sub_module, page_name 等

### 关键 API 端点
- 提交查询：`POST /olap/api/v2/statement/query`
- 轮询状态：`POST /olap/api/v2/statement/getStatusAndLog`（params 参数）
- 获取结果：`POST /olap/api/v2/statement/fetchResult`（**params 参数，不是 json body**）
- 关闭查询：`POST /olap/api/v2/statement/close`（params 参数）
